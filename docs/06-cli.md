# Operator CLI 设计

> `quicktun` 二进制：operator 在自己笔记本上的入口。也是 SSH `ProxyCommand` 用来打 TCP 隧道的工具。

## 1. 命令一览

```
quicktun login                              登录控制面
quicktun logout                             撤销当前 session
quicktun whoami                             查看当前 operator + 可访问 project

quicktun project ls                         列出可访问 project
quicktun project create <slug>              创建 project (admin)
quicktun project rm <slug>                  删除 project (admin)

quicktun site ls --project=<slug>           列出 site
quicktun site add <site-slug> --project=<p> 创建 site，输出安装命令
quicktun site rm <site-slug> --project=<p>
quicktun site rotate-token <site-slug> --project=<p>

quicktun service ls --site=<p>/<s>          列出 service
quicktun service add --site=<p>/<s> --name=<svc> --target=127.0.0.1:22
quicktun service rm   --site=<p>/<s> --name=<svc>

quicktun ssh-config                         生成 SSH config 片段（写入 ~/.ssh/config.d/quicktun）
quicktun status                             各 site 在线状态
quicktun audit --project=<slug> --tail=50

# 内部命令（被 SSH ProxyCommand 调用，不是给人用的）
quicktun connect <site-or-service-name>     建立 TCP 隧道，stdin/stdout 透传
quicktun forward <service> --local=<port>   本地端口转发（给 RDP/DB 等非 SSH 用）
```

## 2. 配置文件

```
~/.quicktun/
├── config.yaml              # 控制面地址、TLS 选项
├── token                    # access_token（mode 0600）
└── sessions/                # 备用：多账号切换（Phase 2）
```

`config.yaml`：

```yaml
control_plane:
  endpoint: relay.example.com:9443
  tls:
    insecure_skip_verify: false
    server_ca: ~/.quicktun/server-ca.pem  # 可选

defaults:
  ssh_user: qt-ops
```

## 3. login 流程

```bash
$ quicktun login
Email: zhang@example.com
Password: ******
✓ Logged in as zhang@example.com
  Token expires: 2026-04-28 22:00 (8h)
  Accessible projects: clinic-network, factory-iot
  
Run 'quicktun ssh-config' to refresh your SSH config.
```

实现：
1. 读 `~/.quicktun/config.yaml` 拿 endpoint
2. gRPC 调 `AuthService.Login`
3. 写 token + expire_time 到 `~/.quicktun/token`（0600）
4. 自动调 `RefreshSSHConfig()`（见下）

## 4. ssh-config 生成

`quicktun ssh-config` 会**全量重写** `~/.ssh/config.d/quicktun`，主 SSH config 需要 include：

```
# ~/.ssh/config
Include config.d/quicktun
```

生成内容示例：

```
# AUTO-GENERATED by quicktun. DO NOT EDIT.
# Project: clinic-network
# Generated at: 2026-04-28T14:00:00+08:00

# Site: hospital-shanghai (跳板机本身)
Host hospital-shanghai
    HostName relay.example.com
    Port 443
    User qt-ops
    ProxyCommand quicktun connect projects/clinic-network/sites/hospital-shanghai/services/ssh
    ServerAliveInterval 30

# Internal target via bastion
Host hospital-shanghai-dc
    HostName 192.168.10.50
    User admin
    ProxyJump hospital-shanghai

Host hospital-shanghai-his
    HostName 192.168.10.100
    User administrator
    ProxyJump hospital-shanghai
    LocalForward 13389 192.168.10.100:3389  # for RDP
```

**生成规则**：
- 每个 service `name=ssh, target=127.0.0.1:22` 的 site → 一个直连 Host 条目（`<site-slug>`）
- 每个 service target 在 LAN（非 127.0.0.1）→ 一个 `ProxyJump` 条目（`<site-slug>-<service-name>`）
- TCP 服务（22/SSH）用 ProxyJump；非 SSH 服务（3389/RDP）加 LocalForward 注释提示

## 5. ProxyCommand: `quicktun connect`

```
$ ssh hospital-shanghai
  ↓ ProxyCommand quicktun connect projects/clinic-network/sites/hospital-shanghai/services/ssh
  
quicktun connect:
  1. 读 ~/.quicktun/token
  2. tls.Dial relay.example.com:443
  3. 发送首帧:
     {
       "version": 1,
       "token": "<access_token>",
       "target": "projects/clinic-network/sites/hospital-shanghai/services/ssh"
     }
  4. 等待服务端返回 OK 帧（或错误码）
  5. io.Copy 双向：os.Stdin <-> conn, conn <-> os.Stdout
  6. ssh 协议在 stdin/stdout 上跑
```

错误码：
- `unauthenticated` → "Token expired. Run 'quicktun login' again."
- `permission_denied` → "You don't have access to this site."
- `not_found` → "Site not found or not registered."
- `unavailable` → "Site is offline (last seen 3 days ago)."

## 6. forward: 非 SSH 协议

```bash
$ quicktun forward projects/clinic-network/sites/hospital-shanghai/services/his --local=13389
Listening on localhost:13389 → his (target 192.168.10.100:3389) via hospital-shanghai
Press Ctrl+C to stop.

# 另一个终端：
$ mstsc /v:localhost:13389
```

实现：本地起 net.Listener，每个 accept 出来的 conn 都跑一遍 `quicktun connect` 的 TLS 握手 + splice。

## 7. CLI 框架

用 [spf13/cobra](https://github.com/spf13/cobra) + [spf13/viper](https://github.com/spf13/viper)：

```go
var rootCmd = &cobra.Command{
    Use:   "quicktun",
    Short: "Multi-site remote access tool",
}

func init() {
    rootCmd.AddCommand(loginCmd, logoutCmd, projectCmd, siteCmd, serviceCmd,
                       sshConfigCmd, statusCmd, connectCmd, forwardCmd, auditCmd)
}
```

## 9. 错误友好

不要把 gRPC 错误原样吐给用户：

```go
func PrintErr(err error) {
    s, ok := status.FromError(err)
    if !ok { fmt.Fprintln(os.Stderr, err); return }
    
    switch s.Code() {
    case codes.Unauthenticated:
        fmt.Fprintln(os.Stderr, "✗ Not logged in or token expired.")
        fmt.Fprintln(os.Stderr, "  Run: quicktun login")
    case codes.PermissionDenied:
        fmt.Fprintln(os.Stderr, "✗ Permission denied.")
        fmt.Fprintf(os.Stderr,  "  %s\n", s.Message())
    // ...
    default:
        fmt.Fprintf(os.Stderr, "✗ %s: %s\n", s.Code(), s.Message())
    }
    os.Exit(1)
}
```

## 10. AI 工具集成

只要 `quicktun ssh-config` 写好 `~/.ssh/config.d/quicktun`，下面工具开箱即用：

| 工具 | 用法 |
|---|---|
| **Cursor / VSCode Remote-SSH** | 选择 host `hospital-shanghai` 即可 |
| **Claude Code** | `ssh hospital-shanghai` 进去后启动 `claude` |
| **rsync / scp** | `scp file.txt hospital-shanghai:~/` |
| **mstsc (Windows RDP)** | `quicktun forward ... --local=13389` 后 `mstsc localhost:13389` |
