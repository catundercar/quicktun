# quicktun Linux Deployment Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the deployment artifacts an operator needs to install quicktun on Linux. Two host types:

1. **Control-plane host** — runs `quicktun-server` (gRPC + grpc-gateway) and `quicktun-authproxy` (CONNECT gateway). Optionally fronted by nginx for TLS termination on `:443`.
2. **Agent host** (bastion) — runs `quicktun-agent` to expose its local services through the relay.

**Out of scope:**
- HA / multi-replica control plane (Plan 11+).
- Windows agent (Plan 7.5).
- Container images / Helm charts (Plan 11+).
- Automatic Let's Encrypt issuance (operator runs `certbot` themselves; we ship the nginx config that uses the cert).

---

## Design

### Filesystem layout (control plane)

```
/usr/local/bin/
├── quicktun-server
├── quicktun-authproxy
├── quicktun                  (operator CLI — only on admin's workstation, not the server host)

/etc/quicktun/
├── server.yaml               control-plane config
├── authproxy.yaml            auth-proxy config

/var/lib/quicktun/
├── quicktun.db               SQLite (server's primary, authproxy's read-only)
├── relays/                   per-project rendered rathole configs

/var/log/quicktun/            (zap+lumberjack writes here)
├── server.log
├── authproxy.log

/etc/systemd/system/
├── quicktun-server.service
├── quicktun-authproxy.service
```

A dedicated system user `quicktun` owns `/etc/quicktun`, `/var/lib/quicktun`, `/var/log/quicktun`. Both services run as that user. `nginx` runs as its own user and only needs read access to `/etc/letsencrypt/live/<domain>/`.

### Filesystem layout (agent host)

```
/usr/local/bin/
├── quicktun-agent
├── rathole                   (operator installs separately — out of our scope)

/etc/quicktun/
├── agent.yaml                pasted from `quicktun site get-install-command` output

/var/lib/quicktun-agent/
├── rathole-client.toml       rendered by agent at runtime

/etc/systemd/system/
├── quicktun-agent.service
```

System user `quicktun-agent`.

### TLS

Phase 1 expects nginx in front of auth-proxy (TCP stream module) for TLS. The nginx template uses `stream` with `ssl_certificate` paths under `/etc/letsencrypt/live/<domain>/`. Operators run certbot on first install. We do NOT auto-issue certs.

TLS termination on **gRPC** port (control plane) is left to the operator: either a separate nginx http2 server block, or grpc-gateway TLS via the existing `Backend.TLSCertFile/TLSKeyFile` config (which we may need to add — verify in code).

For Phase 1 simplicity, we recommend operators terminate gRPC TLS at nginx (server_name + http2). We provide the nginx snippet.

## Task 0: systemd unit files

Three unit files. All use `Type=simple`, `Restart=on-failure`, `RestartSec=5`, `RestartLimitBurst=10` to avoid thrash.

### `deploy/systemd/quicktun-server.service`

```ini
[Unit]
Description=quicktun control plane
After=network-online.target
Wants=network-online.target
Documentation=https://github.com/tulip/quicktun

[Service]
Type=simple
User=quicktun
Group=quicktun
ExecStartPre=/usr/local/bin/quicktun-server migrate --config /etc/quicktun/server.yaml
ExecStart=/usr/local/bin/quicktun-server serve --config /etc/quicktun/server.yaml
Restart=on-failure
RestartSec=5s
StartLimitBurst=10
StartLimitIntervalSec=60s

# Hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
PrivateTmp=true
ReadWritePaths=/var/lib/quicktun /var/log/quicktun
CapabilityBoundingSet=
AmbientCapabilities=

[Install]
WantedBy=multi-user.target
```

### `deploy/lib.sh`

Shared helpers used by install scripts:

```bash
# deploy/lib.sh — shared bash helpers
# shellcheck shell=bash

set -euo pipefail

log()  { printf '\033[1;34m==>\033[0m %s\n' "$*"; }
warn() { printf '\033[1;33m==>\033[0m %s\n' "$*" >&2; }
fail() { printf '\033[1;31m==>\033[0m %s\n' "$*" >&2; exit 1; }

require_root() {
    [[ $EUID -eq 0 ]] || fail "this script must run as root (use sudo)"
}

require_linux() {
    [[ "$(uname -s)" == "Linux" ]] || fail "this script only runs on Linux"
}

ensure_user() {
    local user=$1 home=$2
    id -u "$user" >/dev/null 2>&1 && return 0
    log "creating system user $user"
    useradd --system --no-create-home --home "$home" --shell /usr/sbin/nologin "$user"
}

ensure_dir() {
    local dir=$1 owner=$2 mode=$3
    mkdir -p "$dir"
    chown "$owner:$owner" "$dir"
    chmod "$mode" "$dir"
}

ensure_file_unchanged_or_prompt() {
    local existing=$1 incoming=$2
    if [[ -f "$existing" ]]; then
        if ! cmp -s "$existing" "$incoming"; then
            warn "$existing differs from packaged version; not overwriting"
        fi
        return 0
    fi
    cp "$incoming" "$existing"
}
```

### `deploy/install-server.sh`

```bash
#!/usr/bin/env bash
# Installs quicktun-server + quicktun-authproxy on a Linux host.
# Usage: sudo ./install-server.sh [--from-source]
# Assumes binaries at ./bin/ (after `make build`) or downloads from GitHub release.

set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
# shellcheck source=lib.sh
source "$SCRIPT_DIR/lib.sh"

require_root
require_linux

REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
SERVER_BIN="$REPO_ROOT/bin/quicktun-server"
AUTHPROXY_BIN="$REPO_ROOT/bin/quicktun-authproxy"

[[ -x "$SERVER_BIN" ]] || fail "missing $SERVER_BIN — run 'make build' first"
[[ -x "$AUTHPROXY_BIN" ]] || fail "missing $AUTHPROXY_BIN — run 'make build' first"

# 1. System user + dirs
ensure_user quicktun /var/lib/quicktun
ensure_dir /etc/quicktun       quicktun 0750
ensure_dir /var/lib/quicktun   quicktun 0750
ensure_dir /var/log/quicktun   quicktun 0750
ensure_dir /var/lib/quicktun/relays quicktun 0750

# 2. Install binaries
log "installing binaries to /usr/local/bin"
install -m 0755 "$SERVER_BIN"    /usr/local/bin/quicktun-server
install -m 0755 "$AUTHPROXY_BIN" /usr/local/bin/quicktun-authproxy

# 3. Install systemd units
log "installing systemd units"
install -m 0644 "$SCRIPT_DIR/systemd/quicktun-server.service"    /etc/systemd/system/
install -m 0644 "$SCRIPT_DIR/systemd/quicktun-authproxy.service" /etc/systemd/system/
systemctl daemon-reload

# 4. Install example configs (only if missing)
log "installing example configs (if not present)"
ensure_file_unchanged_or_prompt /etc/quicktun/server.yaml    "$SCRIPT_DIR/etc/server.yaml.example"
ensure_file_unchanged_or_prompt /etc/quicktun/authproxy.yaml "$SCRIPT_DIR/etc/authproxy.yaml.example"
chown quicktun:quicktun /etc/quicktun/server.yaml /etc/quicktun/authproxy.yaml
chmod 0640 /etc/quicktun/server.yaml /etc/quicktun/authproxy.yaml

# 5. Run migrations (server's ExecStartPre will too, but doing it here gives
#    immediate feedback if the config is wrong)
log "running migrations"
sudo -u quicktun /usr/local/bin/quicktun-server migrate --config /etc/quicktun/server.yaml

# 6. Prompt to create admin operator (only if no operator exists)
if ! sudo -u quicktun /usr/local/bin/quicktun-server admin list-operators \
        --config /etc/quicktun/server.yaml 2>/dev/null | grep -q .; then
    log "no operators yet; create the first admin"
    read -rp "Admin email: " ADMIN_EMAIL
    read -rsp "Admin password: " ADMIN_PASSWORD; echo
    sudo -u quicktun /usr/local/bin/quicktun-server admin create-operator \
        --config /etc/quicktun/server.yaml \
        --email "$ADMIN_EMAIL" --password "$ADMIN_PASSWORD" --admin
fi

# 7. Enable + start services
log "enabling + starting services"
systemctl enable --now quicktun-server.service
sleep 1
systemctl enable --now quicktun-authproxy.service

log "Done. Edit /etc/quicktun/server.yaml + authproxy.yaml as needed, then:"
log "  systemctl status quicktun-server quicktun-authproxy"
log "  journalctl -u quicktun-server -f"
log
log "Next: install nginx (see deploy/nginx/quicktun.conf.tmpl) and run certbot."
```

Verify the `admin list-operators` subcommand exists. If not, use a different "is there an operator" check (e.g., check the DB directly via `sqlite3`). Read `cmd/quicktun-server/cmd_admin*.go` to confirm.

If `admin list-operators` doesn't exist yet, add it as a small follow-up:

## Task 2: nginx config template

### `deploy/nginx/quicktun.conf.tmpl`

```nginx
# /etc/nginx/sites-available/quicktun.conf
# Replace CONTROL_DOMAIN and RELAY_DOMAIN.
# Run certbot: certbot --nginx -d CONTROL_DOMAIN -d RELAY_DOMAIN

# ---- gRPC control plane (HTTP/2 + TLS on :443 for CONTROL_DOMAIN) ----
server {
    listen 443 ssl http2;
    server_name CONTROL_DOMAIN;

    ssl_certificate     /etc/letsencrypt/live/CONTROL_DOMAIN/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/CONTROL_DOMAIN/privkey.pem;
    ssl_protocols       TLSv1.2 TLSv1.3;

    location / {
        grpc_pass grpc://127.0.0.1:9090;
        # Pass-through gRPC headers; nginx 1.13.10+
        grpc_set_header Host $host;
    }
}

# ---- HTTP gateway (REST → gRPC) for human/curl operators ----
server {
    listen 443 ssl;
    server_name api.CONTROL_DOMAIN;

    ssl_certificate     /etc/letsencrypt/live/CONTROL_DOMAIN/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/CONTROL_DOMAIN/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:9091;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $remote_addr;
    }
}

# ---- Auth-proxy on RELAY_DOMAIN:443 (TLS termination → :8443 plain CONNECT) ----
# Uses nginx stream module; ensure nginx is built with --with-stream.
stream {
    upstream quicktun_authproxy {
        server 127.0.0.1:8443;
    }

    server {
        listen 443 ssl;
        server_name RELAY_DOMAIN;

        ssl_certificate     /etc/letsencrypt/live/RELAY_DOMAIN/fullchain.pem;
        ssl_certificate_key /etc/letsencrypt/live/RELAY_DOMAIN/privkey.pem;
        ssl_protocols       TLSv1.2 TLSv1.3;

        proxy_pass quicktun_authproxy;
        proxy_protocol off;
    }
}
```

**Caveat**: nginx `stream` block must be at the top-level config, NOT inside `http` block. Operators may need to put this in `/etc/nginx/nginx.conf` directly or via `include /etc/nginx/streams-enabled/*.conf;`.

Document this in the deploy README. The .conf.tmpl file is the source of truth that gets split between `sites-available/` (http servers) and `streams-enabled/` (stream block).

Actually — make this two files:

- `deploy/nginx/quicktun-http.conf` — the two http server blocks (control + http gateway)
- `deploy/nginx/quicktun-stream.conf` — the stream block (auth-proxy)

Operators include each in the appropriate place. README explains.

### Skip auto-install of nginx config

The install-server.sh **does NOT** install nginx config — it's too distro-specific and can clash. Operators copy the templates manually. Document this in deploy/README.md.

## Task 3: agent install + GetSiteInstallCommand polish

### `deploy/install-agent.sh`

```bash
#!/usr/bin/env bash
# Installs quicktun-agent on a bastion host.
# Usage:
#   sudo ./install-agent.sh --token <RAW_TOKEN> \
#                            --control-endpoint <host:port> \
#                            --auth-proxy <host:port>

set -euo pipefail
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
# shellcheck source=lib.sh
source "$SCRIPT_DIR/lib.sh"

require_root
require_linux

TOKEN=
CONTROL_ENDPOINT=
AUTH_PROXY=
TLS_INSECURE=false

while [[ $# -gt 0 ]]; do
    case "$1" in
        --token) TOKEN=$2; shift 2 ;;
        --control-endpoint) CONTROL_ENDPOINT=$2; shift 2 ;;
        --auth-proxy) AUTH_PROXY=$2; shift 2 ;;
        --tls-insecure) TLS_INSECURE=true; shift ;;
        *) fail "unknown arg: $1" ;;
    esac
done

[[ -n "$TOKEN" && -n "$CONTROL_ENDPOINT" && -n "$AUTH_PROXY" ]] || \
    fail "--token, --control-endpoint, --auth-proxy are required"

REPO_ROOT="$(cd "$SCRIPT_DIR/.." && pwd)"
AGENT_BIN="$REPO_ROOT/bin/quicktun-agent"
[[ -x "$AGENT_BIN" ]] || fail "missing $AGENT_BIN — run 'make build' first"

ensure_user quicktun-agent /var/lib/quicktun-agent
ensure_dir /etc/quicktun       quicktun-agent 0750
ensure_dir /var/lib/quicktun-agent quicktun-agent 0750
ensure_dir /var/log/quicktun-agent quicktun-agent 0750

install -m 0755 "$AGENT_BIN" /usr/local/bin/quicktun-agent
install -m 0644 "$SCRIPT_DIR/systemd/quicktun-agent.service" /etc/systemd/system/
systemctl daemon-reload

# Render agent.yaml
log "writing /etc/quicktun/agent.yaml"
cat > /etc/quicktun/agent.yaml <<EOF
control_endpoint: $CONTROL_ENDPOINT
auth_proxy_endpoint: $AUTH_PROXY
token: $TOKEN
state_dir: /var/lib/quicktun-agent
rathole_binary: /usr/local/bin/rathole
rathole_args: ["--client"]
tls_insecure: $TLS_INSECURE
EOF
chown quicktun-agent:quicktun-agent /etc/quicktun/agent.yaml
chmod 0600 /etc/quicktun/agent.yaml

systemctl enable --now quicktun-agent.service
log "Done. Check: systemctl status quicktun-agent && journalctl -u quicktun-agent -f"
```

This requires injecting the control-endpoint + auth-proxy-endpoint into the SiteService struct. Read existing config plumbing — `s.relayAddr` exists; add similar threading for `controlEndpoint` and `authProxyEndpoint`.

## Task 5: lint + final verification

### `scripts/lint-deploy.sh`

```bash
#!/usr/bin/env bash
set -euo pipefail
ROOT="$(cd "$(dirname "$0")/.." && pwd)"
cd "$ROOT"

echo "=== bash -n ==="
for f in deploy/*.sh deploy/lib.sh; do
    [[ -f "$f" ]] && bash -n "$f" && echo "OK $f"
done

echo "=== shellcheck (warnings only) ==="
if command -v shellcheck >/dev/null; then
    shellcheck deploy/*.sh deploy/lib.sh || true
else
    echo "shellcheck not installed; skipping"
fi

echo "=== systemd-analyze verify ==="
if command -v systemd-analyze >/dev/null; then
    for f in deploy/systemd/*.service; do
        # systemd-analyze can't fully verify a unit referencing missing user/group/binaries.
        # Use --recursive=no and accept warnings.
        systemd-analyze verify "$f" 2>&1 || true
    done
else
    echo "systemd-analyze not installed; skipping"
fi

echo "=== nginx -t (smoke parse only) ==="
if command -v nginx >/dev/null; then
    # nginx requires a full config tree; do a syntax-only check.
    for f in deploy/nginx/*.conf; do
        nginx -T -c "$f" 2>&1 | head -1 || true
    done
else
    echo "nginx not installed; skipping"
fi

echo "=== done ==="
```
